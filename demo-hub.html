Experience assets, guided walkthroughs, and reference links

Key Points to Consider

  • These assets are only compatible with Platform. Compatibility with apps such as CS, PGP etc. is on the roadmap.
  • Blueprint automations in the DemoX library can sometimes become outdated due to the biweekly releases from the Blueprint team.
  • This is not a fully built-out application — only a percentage of the app has been developed to allow end-to-end execution for individual case types.
  • These assets are deployed on sandbox environments, so there may be latency issues.

Lessons Learnt

  • If part of a presentation — Allow more than an hour if a Blueprint demo is also on the agenda. More info ▾
    The demo consists of two parts: ideation and the actual software with the omni-channel experience. There is an overlap between a few feature functions, so sufficient time must be allocated to cover both parts effectively.
  • If used as a leave-behind — Book an hour of guided walkthrough with participants. More info ▾
    This ensures participants are familiar with the navigation and know where to click. This session is best conducted in person, with the SC and DemoX team facilitating together.
  • Preparation is key — Factor in time to prepare before the demo. More info ▾
    Transitions and handovers between presenters are important. If you are adding feature functions on top of the Blueprint, both development time and preparation time must be factored into your planning.
